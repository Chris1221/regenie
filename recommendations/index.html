Recommendations for UK Biobank analysis

regenie is ideally suited for large-scale analyses such as 500K UK Biobank data, where records are available for thousands of phenotypes.

We provide below a few guidelines on how to perform such analysis.

Pre-processing

We will first go over important steps to consider before running regenie.

Selection of traits

regenie can perform whole genome regression on multiple traits at once and this is where higher computational gains are obtained. As different traits can have distinct missing patterns, regenie uses an imputation scheme to handle missing data, and real data applications have shown that it gives nearly identical results as discarding missing observations when analyzing each trait separately.

Hence, we recommend to analyze traits in groups that have similar missingness patterns. We are working on releasing a tool that generate these groupings to facilitate this step.

Step 1

We recommend to run regenie using multi-threading (4+ threads) which will decrease the overall runtime of the program. As this step can be quite memory intensive (due to storing block predictions), we recommend to use option --loocv, where the number of phenotypes analyzed will determine how much disk space is required (e.g. around 10GB per phenotype assuming there are 500,000 variants split in blocks of 1,000).

Step 2

We recommend to run regenie using multi-threading (8+ threads) which will decrease the overall runtime of the program.
